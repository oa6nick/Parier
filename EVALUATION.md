## 2. СЛОЖНОСТЬ РАБОТЫ

### 2.1 Общая оценка: **средняя** (6/10)

| Аспект | Сложность | Комментарий |
|--------|-----------|-------------|
| **Фронтенд** | Низкая | Next.js, React, стандартные паттерны |
| **Бэкенд** | Средняя | Go, Gin, GORM — типичный стек |
| **Auth** | Средняя | Keycloak OIDC, интеграция с NextAuth |
| **Интеграция** | Средняя | Маппинг схем, CORS, прокси |
| **Бизнес-логика** | Средняя | P2P ставки, пулы, расчёт выигрышей |
| **Локализация** | Низкая | next-intl + JSON, опционально БД |
| **Кошелёк / токены** | Средняя | Сложнее, чем простой CRUD |

### 2.2 Риски и сложные места

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Расхождение схем фронт/бэк | Высокая | Среднее | Контракт API (OpenAPI), маппинг-слой |
| Keycloak + NextAuth | Средняя | Высокое | Документация, тестовый realm |
| Модель токенов в БД | Средняя | Среднее | Отдельная миграция, согласование |
| Локализация (JSON vs БД) | Низкая | Низкое | Оставить JSON на фронте |
| Производительность при росте | Низкая | Среднее | Индексы, пагинация уже есть |

### 2.3 Сложность по этапам

| Этап | Сложность | Время |
|------|-----------|-------|
| Auth (Keycloak ↔ NextAuth) | 7/10 | 1–2 дня |
| API-прокси, CORS | 3/10 | 0.5 дня |
| Маппинг CreateBet | 5/10 | 1 день |
| Лента из API | 4/10 | 1 день |
| Кошелёк | 6/10 | 2–3 дня |
| Принятие ставок | 6/10 | 2 дня |
| Админка credit-tokens | 5/10 | 1 день |

## 3. ОЦЕНКА ВЫБРАННОГО СТЕКА

### 3.1 Общая оценка: **хороший выбор** (8/10)

Стек современный, уместный для P2P-платформы. Ниже — по компонентам.

### 3.2 Фронтенд

| Технология | Оценка | Плюсы | Минусы |
|------------|--------|-------|--------|
| **Next.js 15** | ✅ 9/10 | App Router, SSR, i18n, Vercel | Переход на 16 — возможны breaking changes |
| **React 19** | ✅ 8/10 | Актуальный, concurrent features | Меньше примеров, чем для 18 |
| **TypeScript** | ✅ 9/10 | Типизация, меньше ошибок | — |
| **Tailwind** | ✅ 9/10 | Быстрая разработка, адаптив | — |
| **next-intl** | ✅ 8/10 | Удобная i18n, routing | — |
| **Без Redux/Zustand** | ⚠️ 6/10 | Проще | Для глобального состояния (auth, баланс) может понадобиться |

**Рекомендация:** Добавить Zustand или React Context для auth/balance — легче, чем Redux.

### 3.3 Бэкенд

| Технология | Оценка | Плюсы | Минусы |
|------------|--------|-------|--------|
| **Go + Gin** | ✅ 9/10 | Быстрый, мало ресурсов, много библиотек | — |
| **GORM** | ✅ 8/10 | ORM, миграции | N+1 при неаккуратных запросах |
| **PostgreSQL** | ✅ 9/10 | Надёжность, JSON, расширения | — |
| **Keycloak** | ✅ 8/10 | OAuth2, роли, self-hosted | Сложнее настройка, чем Auth0 |
| **MinIO** | ✅ 9/10 | S3-совместимость, self-hosted | — |
| **Liquibase** | ✅ 8/10 | Версионирование миграций | Два инструмента (Liquibase + GORM) |

**Рекомендация:** Liquibase для схемы — ок. GORM для моделей — ок. Не смешивать миграции в GORM и Liquibase.

### 3.4 Инфраструктура

| Технология | Оценка | Плюсы | Минусы |
|------------|--------|-------|--------|
| **Docker Compose** | ✅ 9/10 | Всё в одном стеке, dev | Для prod — лучше K8s/ECS |
| **n8n** | ⚠️ 6/10 | Автоматизация, AI | Для MVP — избыточно |
| **PostGIS** | ⚠️ 5/10 | Геоданные | Нет гео в текущем функционале — лишний объём |

**Рекомендация:** n8n и PostGIS можно отключить или не использовать в MVP.

### 3.5 Что можно улучшить

| Область | Текущее | Рекомендация |
|---------|---------|--------------|
| **State management** | Нет глобального | Zustand или Context для auth |
| **API-клиент** | fetch напрямую | axios или fetch с обёрткой + типы из OpenAPI |
| **Тесты** | Vitest, unit | Добавить E2E (Playwright) для критичных сценариев |
| **Auth** | Mock vs Keycloak | Выбрать один источник правды |
| **Локализация** | JSON на фронте | Оставить JSON на MVP, БД — позже |

### 3.6 Альтернативы (для сравнения)

| Компонент | Текущий | Альтернатива | Плюс альтернативы |
|-----------|---------|--------------|-------------------|
| Бэкенд | Go + Gin | Node.js + NestJS | Один язык (JS/TS), общие типы |
| Auth | Keycloak | NextAuth + Credentials | Проще |
| БД | PostgreSQL | PostgreSQL | — (выбор верный) |
| Файлы | MinIO | S3 (AWS) | Managed, меньше администрирования |

**Вывод:** Go на бэкенде — хороший выбор для производительности и масштабирования. Связка Next.js + Go — распространённая и рабочая.
